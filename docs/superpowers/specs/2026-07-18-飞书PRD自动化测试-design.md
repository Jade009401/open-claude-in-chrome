# 飞书 PRD → 自动化测试 · 设计文档

> 日期:2026-07-18(v5,含三轮 spec 评审 + 两轮对抗红队)｜ 承载项目:open-claude-in-chrome ｜ 状态:设计已定,待用户复核 → 实施计划
> v5 关键调整(路 A):① MVP 砍成**试点优先 + 止损门槛**,不先建全量 12 组件;② 补**写操作安全边界**(红队实证:目前零护栏,D11 还拆掉了唯一软保护);③ 补**测试数据/前置状态**真空;④ 如实标注"语义校验/失败分类"当前**无代码底座、工作量被低估**。

## 1. 背景与目标

给**测试人员**用:喂规范化飞书提测文档,系统读懂需求、生成可重放脚本、真实执行、产出「通过/失败 + 证据」报告并播报回飞书。构建在 open-claude-in-chrome 上(CDP + Pure Map)。自主度 **L2(AI 生成+执行,人审报告)**。

## 2. 用户与体验

- 使用者:测试人员(非开发)。约束:测试端**零安装门槛、零授权步骤**。
- 触点:浏览器 Claude 侧栏(触发 + 展示)。体验:粘链接 → 点开始 → 看结果。
- 部署:本地每人一套(用自己已登录浏览器;对话/状态天然隔离;无服务器)。

## 3. 范围 —— 试点优先(路 A 核心)

红队实证结论:整套系统 12 个新组件工作量大,且行业数据显示"AI 自愈/自动测试"最常见的死法是**慢性信任流失**(假绿反复→人工复查→省时卖点被吃掉→沦为摆设)。故**不先建全量,先做能证明价值的最小试点**。

### Phase 0 · 试点(先做,带止损门槛)

- **范围**:3–5 个真实业务场景。
- **刻意避开致命区**(让试点便宜、安全、可信):
  - **只做只读断言为主**的场景(打开页面→核对展示是否符合 PRD),写操作最小化。
  - **仅测试环境**(域名白名单,生产一律拒绝,见 §5 安全)。
  - **不上自愈**:锚点失效直接标"不确定"交人审(避开红队指出的"语义校验无代码底座"的坑)。
  - 写操作(如有)**一律真人确认**,不自动执行。
- **目标**:验证「AI 读 PRD → 生成脚本 → 重放 → 判定 → 出证据报告」这条主链在真实场景的**改判率**和**耗时**。
- **止损门槛(先定阈值,用数据决定继续/停)**:
  - 若测试人员对"通过"的**人工改判率 > X%**(建议 20–30%,试点前定死)→ 说明 AI 判定不可托付,停或回炉。
  - 若单场景端到端**耗时 ≥ 测试人员手工验收**→ 省时卖点不成立,停或缩范围。

### Phase 1+ · 完整系统(试点过关才建)

自愈 + 失败分类 + 映射库沉淀 + 便宜模型下沉 + 写操作安全体系 + 一键安装等(§4 目标架构全量)。**试点未过止损门槛不进入本阶段。**

**始终不做:** 修 bug(只报告);跨系统串测;移动端/App。

## 5. 架构 · 关键实现真相 · 安全边界

### 现有代码真相(硬证据,决定工作量估算)

- ✅ `MapEngine.locate`(`host/map/map-engine.mjs`)纯本地函数 → 编排器可 import 直调。
- ⚠️ `map/read/act`(`host/mcp-server.js` → 单例 `nativeHostSocket` → `native-host.js` TCP → `chrome.debugger` CDP,弹"自动化控制"横幅)→ 编排器须同进程或复用 TCP 握手,**非直调库**。
- ⚠️ 飞书 SDK 适配器**须全新重建**:旧 `lark-adapter.js` 及依赖已被 `4792012 回退 daemon` 删除(旧架构废弃物);`@larksuiteoapi` 在 node_modules 但**未声明依赖须补**。
- ⚠️ **红队实证:v4 号称的"语义一致性二次校验"与"失败分类自愈"当前无任何代码底座**:重定位 `resolvePath` 是纯 `querySelector` 链找到即返回,锚点结构(`normalizeAnchor`)**无 role 字段**,roadmap 把"更强失效复核"列为未开始。→ 实现需**新增锚点 schema 字段(建图阶段也要改)+ 比对逻辑 + 专门黄金集**,工作量大,**Phase 0 不做**,Phase 1 单列为大块。

### ⚠️ 写操作安全边界(红队实证:目前零护栏,必须新建)

现有唯一护栏 `confirmed:true` 是调用方自传的布尔值,**无人类交互产生**;无元素白名单、无环境区分。而 D11 把 Claude 踢出循环 → 编排器只能自己硬编码 `confirmed:true`,**等于架构亲手拆掉唯一软保护**。故必须新建:

1. **环境白名单**:配置允许的测试环境域名;**生产/未知域名一律拒绝写操作**(只读放行)。base URL 探路探到白名单外 → 停并报错。
2. **写操作真人确认**:`click/input/select/submit` 等改数据动作,尤其 delete/pay/approve/提交 类,**必须侧栏卡片真人二次确认**,不接受编排器自置 `confirmed:true`。
3. **破坏性动作永不自动重放**:归入"每次都人审",不走确定性自动执行。
4. **自愈不碰写操作**:写操作步骤锚点失效**一律停+人审**,绝不自愈后自动 submit(防误点他人数据行)。

### 组件(🆕=Phase 1 全量;Phase 0 只取只读子集)

`MapEngine.locate`(复用直调)｜`map/read/act` 通道(复用握手)｜host QA 编排器 🆕｜飞书 SDK 轻量客户端 🆕｜脚本生成器 🆕｜脚本重放器 🆕｜**自愈+语义校验+失败分类(无代码底座,大块,Phase 1)** 🆕｜环境白名单+写确认(安全,Phase 0 就要基础版)🆕｜Lark Base 客户端 🆕｜播报器 🆕｜侧栏 UI 🆕｜一键安装器 🆕。

### 冲突与规避

- **工具面锁**:`validateClaudeToolSurface()` 只管 Claude 可见工具 → host 侧编排、不给 Claude 加工具,绕开。
- **单例通道并发 [HIGH·待验证]**:`map/read/act` 共享单例 socket + per-tab `chrome.debugger`,与侧栏会话共用 → 实施前验证进程关系/并发/CDP 横幅影响(§11)。

## 6. 数据流(一次任务)

```
① 触发   侧栏粘链接 → 编排器开跑
② 读文档 [应用身份] 读专用知识库;先探测 table 块,有表格走 blocks 抽取或标"待人工确认",不无脑吃 raw_content
③ 出脚本 查 Lark Base 有无既有脚本(复用/增量);无则文档→锚点化步骤序列(可便宜模型)
④ 定入口 文档入口→查映射库→base URL 探路;探到【环境白名单外】立即停+报错
         首跑探路设【预算/轮次上限】,超限转"提示补映射",不无限重试(控冷启动成本)
⑤ 执行   锚点命中→操作;写操作→【真人确认】才执行,生产域名拒绝
         锚点失效:Phase 0 直接标"不确定"人审;Phase 1 才 分类失败原因→仅高置信 selector 类自愈→语义二次校验→刚自愈过一律高优先级人审
         前置数据/状态缺失(如空列表)→ 标"前置缺失,待人工",不误判成"失败=真 bug"
         每步判 通过/失败/不确定;关键点截图
⑥ 固化   首跑成功脚本+新映射写回 Lark Base(自检过才入库);自愈成功无论首跑与否都回写锚点
⑦ 归档   [应用身份] 每条结果+证据 增量写 Lark Base
⑧ 播报   [应用身份] 结果摘要→机器人发飞书群
⑨ 人审   测试人员看播报/侧栏确认(L2)
```

## 7. 错误处理与边界

**五原则:** 宁停下报错不硬跑｜分清真 bug 与抖动(重试 K 次)｜拿不准标"不确定"人审｜不写脏数据｜边跑边落库。

| 场景 | 处理 |
|---|---|
| 文档读不到/缺入口 | 报错指引;缺入口查映射库→仍无提示补 |
| 表格型需求 | 探测 table 块走 blocks 或标"待人工确认边界" |
| **前置数据/状态缺失**(空列表等) | ⭐ 标"前置缺失,待人工准备",**不误判为失败=真 bug**;提测文档应注明前置准备方式(§10) |
| 功能页找不到 | 探路上限内失败→标"无法定位,跳过",不阻塞、不回写错误映射 |
| 某步执行失败 | 重试 K 次排抖动;仍失败→记"失败"+截图 |
| **写操作** | ⭐ 真人确认才执行;生产/白名单外域名拒绝;破坏性动作永远人审、不自愈 |
| 锚点失效 | Phase 0 标"不确定"人审;Phase 1 分类→仅高置信 selector 自愈+语义校验+强制人审 |
| 便宜模型分类/自评不可靠 | 高置信才自愈,否则一律"不确定";升级强模型叠加历史误判率独立阈值 |
| Lark Base 写失败/脚本 JSON 解析失败/截图上传失败 | 校验+局部重试+明确报错,不静默丢结果/不带病执行 |
| token 过期/限流 | SDK 自动刷新;退避重试 |
| 同一人并发 | 排队一次一个(叠加 §5 单例通道) |

## 8. 结果可信度(证据不对称 · 考虑测试时间)

**原则:通过=轻,失败/不确定=重;截图只在关键点;报告默认摘要可展开。**

- **三态判定**:通过/失败/不确定,拿不准不硬判。
- **证据按需详略**:通过=一行+最终态 1 图;失败/不确定=期望 vs 实际+前后截图+trace。
- **通过也留最小痕迹**:用了哪个锚点/是否自愈/便宜还是强模型/置信度。
- **判定溯源到 PRD 原文**:每条判定标"依据 PRD 哪一句",让人审能分清"文档滞后 vs 真 bug"(防 PRD 写错→AI 照错误预期判绿/误报)。
- **需求覆盖率**:用例 JSON 必填"对应需求 ID";报告给覆盖率,防悄悄漏测还全绿。
- **可复现**:锚点脚本确定性;附"一键重放";自愈路径含非确定性,另标。
- **长期度量(异步)**:抽样人工复核"通过"统误判率、记人工改判率(即 Phase 0 止损指标),回写 Lark Base。
