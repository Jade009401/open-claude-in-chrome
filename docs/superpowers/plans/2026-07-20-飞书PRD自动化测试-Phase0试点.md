# 飞书 PRD 自动化测试 · Phase 0 试点 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development(推荐)或 superpowers:executing-plans 逐任务实现。步骤用 `- [ ]` 复选框追踪。
> 代码注释一律中文。测试框架 `node:test`(项目现有约定,`node --test`)。

**Goal:** 端到端跑通 3–5 个「只读、测试环境」场景(飞书 PRD → 生成锚点脚本 → 确定性重放 → 三态判定 → 证据报告 → 播报飞书 + Lark Base 记录),采集「人工改判率 + 耗时」两个止损指标,用数据决定是否进 Phase 1。

**Architecture:** host 侧独立编排器,`import` 复用 `MapEngine.locate` 纯本地函数,`map/read/act` 复用现有单例 host↔extension 通道;飞书全走官方 SDK `@larksuiteoapi`(应用身份)。**试点刻意避开:自愈、失败分类、生产环境、写操作自动执行。**

**Tech Stack:** Node.js(ESM `.mjs`)、`node:test`、`@larksuiteoapi/node-sdk` v1.70.0、现有 Pure Map(`host/map/*`)。

**依赖规格:** `docs/superpowers/specs/2026-07-18-飞书PRD自动化测试-design.md`(v5,§3 试点范围 / §5 安全 / §11 待验证项)。

**Phase 0 决策补充(本次确认):** 播报到飞书(im)+ 运行记录写 Lark Base 一张表(含"人工改判"列),止损指标从该表计算——结果与指标均在飞书可见。

---

## ⚠️ 前置铁律(每个建设任务都遵守)

- **spike 未过不建下游**:Task 0a/0b 是 gating,任一失败 → 停,把结论写回 spec §11,回炉重估,不硬往下建。
- **只读优先**:Phase 0 场景以"打开页面核对展示"为主;凡涉及写操作 → 走 `safety.mjs` 真人确认,且默认从试点场景里排除。
- **环境白名单**:任何被测 URL 必须在白名单(测试环境域名)内,否则拒绝。
- **不自愈**:锚点失效直接判"不确定",不重定位、不猜。

## 文件结构(Phase 0 全在 open-claude-in-chrome)

| 文件 | 职责 |
|---|---|

| `host/qa/replayer.mjs` | 确定性重放:调 locate/act,执行只读断言,无自愈 |

| `host/qa/tests/*.test.mjs` | 各模块 node:test |
| 侧栏最小触发入口 | 复用现有侧栏,加"粘链接→开始"入口(spike 0b 后按实际接线方式定) |

## Task 0a(spike): 飞书授权读"新建"文档实测

**性质:** 验证,非 TDD。目的:确认应用身份能读专用知识库里**新建**的 PRD(spec §11 头号未决,官方文档未明确)。

**Files:** Create `host/qa/spikes/spike-lark-read.mjs`(一次性脚本,验证后保留作烟测)

- [ ] **Step 1:** 补依赖:`cd host && npm install @larksuiteoapi/node-sdk@1.70.0 --save`(确认写进 `host/package.json` dependencies)。
- [ ] **Step 2:** 管理员在飞书:建专用知识库、授权「自动化测试」应用、放一篇已有 PRD;记录该 doc 的 token。
- [ ] **Step 3:** 写 `spike-lark-read.mjs`:用 `app_id/app_secret` 建 client → `client.docx.document.rawContent({path:{document_id}})` 读已有 PRD → 打印前 500 字。
- [ ] **Step 4:** 运行 `node host/qa/spikes/spike-lark-read.mjs`,确认能读到内容(不是权限错)。
- [ ] **Step 5(关键):** 管理员**新建**一篇 PRD 放进同一知识库,**不做任何额外授权**;把新 doc token 传进脚本再跑。
- [ ] **Step 6:** 判定:
  - ✅ 能读到新文档 → "一次性授权覆盖新文档"成立,继续。
  - ❌ 报 91403/131006 → **停**。把结论写回 spec §11,与用户确认改走"每篇授权"或用户身份 OAuth,再回来。
