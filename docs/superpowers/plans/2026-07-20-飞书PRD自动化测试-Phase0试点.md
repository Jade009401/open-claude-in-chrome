# 飞书 PRD 自动化测试 · Phase 0 试点 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development(推荐)或 superpowers:executing-plans 逐任务实现。步骤用 `- [ ]` 复选框追踪。
> 代码注释一律中文。测试框架 `node:test`(项目现有约定,`node --test`)。

**Goal:** 端到端跑通 3–5 个「只读、测试环境」场景(飞书 PRD → 生成锚点脚本 → 确定性重放 → 三态判定 → 证据报告 → 播报飞书 + Lark Base 记录),采集「人工改判率 + 耗时」两个止损指标,用数据决定是否进 Phase 1。

**Architecture:** host 侧独立编排器,`import` 复用本地定位纯函数(`locateInMap`,或经 `new MapEngine().locate`——最终 import 形态由 Task 0b 定),`map/read/act` 复用现有单例 host↔extension 通道;脚本生成由编排器 **spawn headless `claude -p` CLI**(候选 B′,复用登录态、不引 API key、不加 MCP 工具),已由 Task 0c 实测通过;飞书全走官方 SDK `@larksuiteoapi`(应用身份)。**试点刻意避开:自愈、失败分类、生产环境、写操作自动执行。**

**Tech Stack:** Node.js(ESM `.mjs`)、`node:test`、`@larksuiteoapi/node-sdk` v1.70.0、现有 Pure Map(`host/map/*`)。

**依赖规格:** `docs/superpowers/specs/2026-07-18-飞书PRD自动化测试-design.md`(v5,§3 试点范围 / §5 安全 / §11 待验证项)。

**Phase 0 决策补充(本次确认):** 播报到飞书(im)+ 运行记录写 Lark Base 一张表(含"人工改判"列),止损指标从该表计算——结果与指标均在飞书可见。

## ⚠️ 前置铁律(每个建设任务都遵守)

- **spike 未过不建下游**:Task 0a/0b/0c 是 gating(0c=脚本生成的模型接入;0b 含**截图能力**与**map 生命周期**两项判定),任一失败 → 停,把结论写回 spec §11,回炉重估,不硬往下建。
- **只读优先**:Phase 0 场景以"打开页面核对展示"为主;凡涉及写操作 → 走 `safety.mjs` 真人确认,且默认从试点场景里排除。
- **环境白名单**:任何被测 URL 必须在白名单(测试环境域名)内,否则拒绝。
- **不自愈**:锚点失效直接判"不确定",不重定位、不猜。
